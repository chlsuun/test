1. "Sayno의 가르침" 페르소나 NPC (Unity + LLM API)
공고 연결점: "Gaming Agent 개발", "유저와 상호작용", "몰입도 극대화"

아이디어:

사용자님이 텍스트 마이닝했던 <세이노의 가르침> 데이터를 활용합니다.

유니티(Unity)로 간단한 상점 주인 NPC를 만듭니다.

OpenAI API(또는 로컬 LLM)를 연결하고, 프롬프트 엔지니어링을 통해 **"무조건 세이노처럼 말투와 가치관을 가지고 답변하는 봇"**을 만듭니다.

플레이어가 "아이템 좀 깎아주세요"라고 하면, 단순히 거절하는 게 아니라 *"네가 가난한 이유는 협상을 구걸로 착각하기 때문이야. 제값을 치러라."*라고 답변하게 만듭니다.

어필 포인트: 단순 챗봇이 아니라 게임 속 캐릭터의 성격(Persona)을 LLM으로 구현했다는 점이 게임 회사가 좋아하는 포인트입니다.

2. LLM 기반 경제 뉴스 감성 분석 및 게임 내 시장 예측
공고 연결점: "이론적/경험적 분석", "데이터 기반 의사결정"

아이디어:

사용자님이 관심 있어 하는 금융/경제 데이터를 활용합니다.

실제 경제 뉴스 헤드라인을 크롤링하여 LLM에게 "호재/악재"를 판단하게 시킵니다(Sentiment Analysis).

이 결과를 유니티로 만든 가상의 주식/코인 게임에 연동합니다. 실제 뉴스가 긍정적이면 게임 내 아이템 시세가 오르고, 부정적이면 폭락하는 **"현실 연동형 게임 경제 시스템"**을 구현합니다.

어필 포인트: 데이터 분석 역량과 게임 시스템 기획력을 동시에 보여줄 수 있습니다. MIS 전공의 강점을 살리기 좋습니다.

3. "추리 게임"을 위한 거짓말탐지기 에이전트 (RAG 활용)
공고 연결점: "모델의 추론 능력", "정량적 분석 경험"

아이디어:

범인 찾기 텍스트 게임을 만듭니다.

범인의 알리바이와 증거 목록을 JSON 데이터로 만듭니다.

LLM(ChatGPT 등)에게 이 데이터를 참고(RAG: 검색 증강 생성)하여 플레이어의 질문에 답하게 하되, 범인은 교묘하게 거짓말을 섞어서 대답하도록 프롬프팅합니다.

핵심: 단순히 구현만 하지 말고, *"프롬프트를 어떻게 짰더니 거짓말 성공률이 60%에서 85%로 올랐다"*는 식의 정량적 성능 지표를 보고서로 남기세요. (공고의 '벤치마크 설계', '성능 평가' 항목 저격)
